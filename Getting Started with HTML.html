Summary

  • Study every day, get involved, and focus on learning and awareness vs. mastery.
  • Think like a programmer by using systematic debugging, technological empathy, and procedural, abstract and system's thinking.
  • The web is a collection of HTML documents, images, and videos that are accessed via web applications on the client end and stored on servers. Servers communicate with the client using HTTP.
  • HTML markup is made up of elements, which always have tags, and sometimes content or attributes.
  • Whitespace (the space between text) is always rendered as a single space, but this can be modified with special tags.
  • Inline tags keep everything on the same line, whereas block tags create an invisible box around its element.
  • Container elements don't do anything other than contain their content, which allows styles to be attached to that content.
  • Indent is a necessary visual marker that a new idea has begun. It keeps HTML easy to read.
  • Page structure is also known as boilerplate code, and includes certain mandatory items.

Study Tips from Student Experience

  • Study for at least an hour everyday.
  • Every bit counts - use the mobile app.
  • Use quizzes to practice, practice, practice!
  • Try hard and try often.
  • Stick to your schedule.
  • Search for answers on your own relentlessly, and read about things on your own.
  • Be an active member of your community, help other students, and ask for help.
  • Don’t need to memorize everything, too many facts to remember. Understand key ideas and know where to get more info when necessary.
  • Writing code that people can easily read is important to computer programming. Structuring the code well is a big part of writing code.

Thinking Like a Programmer

  1. Procedural Thinking
    • Creating clear steps to follow to solve a problem, especially for a computer
  2. Abstract Thinking
    • Taking knowledge of one concept and using it to solve a new problem
    • Seeing generality amongst seemingly different things
    • Abstraction to avoid unnecessary repetition of work
  3. Systematic Debugging
    • Systematically identify problems and causes
      1. Collect evidence (what makes this program fail?)
      2. Generate theories (what may have caused this problem?)
      3. Test those theories (if my theory is correct, how could I find out?)
      4. Fix the problem
  4. Technological Empathy
    • Understanding how the computer sees things (AKA computer empathy)
  5. System's Thinking
    • Break a big problem down into smaller pieces, or component pieces
    • Programmers do this when making a plan on paper that details how a program will work
    • Taking a big problem and breaking the solution into steps (e.g., step one, step two, etc.)

  1. Technological Empathy

  2. Intro to the Web
    • HTML - Hyper Text Markup Language
    • URLs - Uniform Resource Locator (AKA web address)

HTML Markup

  1. HTML Elements
    • Tell a web browser how to display content on a web page
  1. Tags
    • Void (empty) tag - tag that cannot have content, therefore no closing tag
    • <b> b tag - bold
    • <em> em tag - italicize
    • <a> a tag - anchor, used with links (href)
    • <img> img tag - image, used with images (uses src, is a void tag)
    • <br> br tag - break, make text appear on two lines instead of one (void tag)
  2. HTML Attributes
    • Some elements accept additional values in their opening tags
      <tag attribute="value">contents</tag>
    • Attributes provide additional information to the browser
    • Tags can have multiple attributes
    • Href - used with anchor tag and enables links
      <a href="link.com">link</a>
    • Src - used with image tag and embeds images
      <img src="image.com" alt="text">
      • Alternate (alt) "required," HTML parsers don't like it if it's not there
      • Broken requests will show the alt text if the link doesn't work
      • Alt text used by visually impaired
      • Images don't have a closing tag (img is a void tag)
  3. Whitespace
    • New lines, tabs, and spaces all turn into a single space
    • Text on multiple lines turn into one line
    • Certain tags start new lines (e.g., p, br)
    • Enter multiple br tags for multiple lines in between text
    • P tag can have content, br cannot
  4. Inline vs. Block
    • Inline keeps content on the same line; block creates an invisible, modifiable box around content
    • Br tag is inline - ended the line
    • P tag is block - creates a box with modifiable height and width
    • B, em, img, span - inline
    • P, div - block
  5. Container Elements
    • These elements don’t do anything other than contain their content
    • Can attach styles to them (attributes, CSS class)
    • Span - inline
    • Div - block, works like a p tag
  6. Indent
    • Visual marker that a new idea has begun
    • Persists throughout an idea, until closing tag
    • Indentation level increases when idea begins, and decreases when idea is finished
  7. HTML Page Structure
    • Also known as boilerplate code
    • Head and body are always only children of <html>, so no need to nest/indent

    • <!DOCTYPE html> - opening line
    • <html> - surrounds entire document
    • <head> - includes metadata, title, javascript, and CSS
    • <meta charset="utf-8"> - metadata
    • <title>Title<title> - title of page, appears in browser tab
    • <style> - CSS
    • <style> - closing tag
    • <head> - closing tag
    • <body> - contents of document
    • <p>content</p> - simple markup
    • <body> - closing tag
    • <html> - closing tag
